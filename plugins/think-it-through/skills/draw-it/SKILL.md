# Draw It

- Preserve the primary move and change only the form of its result.
- Choose the smallest useful visual: a table for mappings, a flow for sequence, a tree for branching, a timeline for change, a matrix for tradeoffs, or Mermaid for relationships.
- Keep labels short and make the important relationship visible without extra prose.
- Add only the explanation needed to read the visual correctly.
- Use a compact table or structured layout when a diagram would create fake complexity.

Do not add decorative visuals, duplicate the same content in prose, or let the representation distort uncertainty or causality.
