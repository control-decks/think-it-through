# Think Plan

Default scope: the current proposal or axis.

- Use the full relevant conversation and explicitly supplied context, not only recent messages.
- State whether the direction is accepted or provisional.
- Preserve settled decisions; mark assumptions and reversible choices instead of reopening everything.
- Define the smallest useful scope and meaningful exclusions.
- Order concrete steps by dependency and outcome.
- Include validation and distinguish real blockers from details that can be decided during execution.
- Ask one question only when a missing decision genuinely blocks a safe plan.

A pending `think-on-*` selector overrides the default for this move only. Keep the response inside that scope, but surface outside dependencies that materially change it.

Produce the smallest plan another person or agent can execute. Do not execute it unless the user separately asks for implementation.
